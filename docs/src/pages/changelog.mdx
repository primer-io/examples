# Changelog

This page documents all notable changes to Composable Checkout. We follow [Semantic Versioning](https://semver.org/) for our releases.

## v0.3.0 - Vault UI Redesign, Payment Method Controls & Architecture Improvements (12-Jun-2025)

### üé® Vault Component Redesign

- **Complete visual overhaul** of the vault manager component following updated designs
- **Responsive layout improvements** with enhanced interaction states and better mobile experience
- **Custom empty state slot**: Override the default empty state with your own content using `<div slot="vault-empty-state">`
- **New showEmptyState option**: Control empty state visibility for drop-in implementations via `vault.showEmptyState: false`
- **Improved accessibility** with better ARIA labels and screen reader support

### ‚ö° Payment Method Control System

- **Individual payment method disabling**: Add `disabled` attribute to any payment component
  ```html
  <primer-card-form disabled></primer-card-form>
  <primer-payment-method type="PAYPAL" disabled></primer-payment-method>
  ```
- **Global payment disabling**: Use `disabledPayments: true` in SDK options to disable all methods at once
- **Enhanced visual states**: Consistent disabled styling across all payment methods using design tokens
- **iframe-based payment support**: Advanced overlay technique for disabling Google Pay, Apple Pay, and PayPal buttons

### üèóÔ∏è Build Improvements

- **Improved bundling pipeline**: Enhanced compilation process with better tree-shaking and reduced bundle sizes
- **Component type fixes**: Fixed missing Lit class types that could cause components to not be recognized as HTML elements in native JavaScript
- **Enhanced developer experience**: Better IntelliSense with explicit interface definitions

### üîß Bug Fixes

- **Fixed PayPal disappearing issue**: Resolved a bug where PayPal buttons would disappear in some setups due to insufficient space allocation
- **Payment method height stabilization**: Replaced dynamic CSS variables with constants to ensure consistent display

### ‚ö†Ô∏è Breaking Changes

- **Stricter TypeScript typing**: More precise type definitions may reveal previously ignored TypeScript errors in your codebase

### Example Usage

```html
<!-- Custom vault empty state -->
<primer-vault-manager>
  <div slot="vault-empty-state">
    <p>No saved payment methods</p>
    <button>Add a payment method</button>
  </div>
</primer-vault-manager>

<!-- Disabled payment methods -->
<primer-card-form disabled></primer-card-form>
<primer-payment-method type="KLARNA" disabled></primer-payment-method>
```

```javascript
// SDK options for vault and global disabling
{
  vault: {
    enabled: true,
    showEmptyState: false
  },
  disabledPayments: true
}
```

## v0.2.0 - Event System Overhaul & PrimerJS API Introduction (19-May-2025)

### ‚ö†Ô∏è Breaking Changes

- **Event namespace changes**: Renamed all DOM events to follow the `primer:*` namespace format:
  - `primer-state-changed` ‚Üí `primer:state-change`
  - `primer-payment-methods-updated` ‚Üí `primer:methods-update`
  - `primer-checkout-initialized` ‚Üí `primer:ready`
  - `primer-card-submit-success` ‚Üí `primer:card-success`
  - `primer-card-submit-errors` ‚Üí `primer:card-error`
  - `primer-card-network-change` ‚Üí `primer:card-network-change`
- **Removed legacy events**: `primer-oncheckout-complete` and `primer-oncheckout-failure` events
- **New PrimerJS class**: Replaced direct access to the headless SDK instance
- **Unified callbacks**: Consolidated payment success and failure callbacks into a single `onPaymentComplete` callback

### ‚ú® New Features

- **PrimerJS class**: Introduced type-safe API surface with:
  - `refreshSession()` method for refreshing client session
  - `getPaymentMethods()` method for retrieving available payment methods
  - `onPaymentStart` callback for payment process initiation
  - `onPaymentPrepare` callback with handler for intercepting payment creation
  - `onPaymentComplete` unified callback for payment completion results

### üìö Documentation

- **Migration guide**: Added [migration guide](documentation/migration-guides/v01-to-v02) for transitioning from v0.1.x to v0.2.x
- **Updated documentation**: Refreshed examples and API reference documentation

### ‚ú® Features

- **üéØ Composable Card Form** with layout customizations and interactive validation
- **üí≥ Payment Method Support**:
  - Initial beta release for PayPal support

- **üåç Localization Support**: Translations for all locales currently supported in SDK v2
- **‚ö° State Management**: Event dispatching and state handling across components
- **üé® Styling API**: Comprehensive CSS variables system for customizing the checkout appearance

### üß© Components

- `<primer-checkout>`: Main container for all checkout components, handles SDK initialization
- `<primer-main>`: Optional container for organizing payment methods and checkout states
- `<primer-payment-method>`: Renders different payment method interfaces based on configuration
- `<primer-card-form>`: Container for card input components with validation
- `<primer-input-card-number>`: Secure input for card numbers with network detection
- `<primer-input-card-expiry>`: Secure input for card expiration dates
- `<primer-input-cvv>`: Secure input for card verification codes
- `<primer-input-card-holder-name>`: Input for cardholder name
- `<primer-card-form-submit>`: Submit button component for card forms
- `<primer-button>`: Versatile button component for actions
- `<primer-input>`: Base input component for text and other data
- `<primer-input-label>`: Label component for form inputs
- `<primer-input-error>`: Error message component with validation state
- `<primer-input-wrapper>`: Container component for input, label and error elements
